Upgrade the /api/chat endpoint to support streaming responses using OpenAI streaming.

GOAL:
Make AI responses stream token-by-token to the frontend instead of returning full JSON at once.

========================================
BACKEND UPDATE (server/index.ts)
========================================

1. Modify POST /api/chat route.

2. Instead of returning res.json({ reply }), implement streaming using:

- OpenAI chat.completions with stream: true
- Use for await (...) to iterate over chunks
- Use res.write() to send tokens progressively
- End with res.end()

3. Set proper headers before streaming:

res.setHeader("Content-Type", "text/plain");
res.setHeader("Transfer-Encoding", "chunked");

4. Example structure:

const completion = await openai.chat.completions.create({
  model: "gpt-4o-mini",
  messages,
  stream: true,
});

for await (const chunk of completion) {
  const content = chunk.choices[0]?.delta?.content;
  if (content) {
    res.write(content);
  }
}

res.end();

5. Wrap everything in try/catch.
6. On error, return res.status(500).end("Streaming error");

FRONTEND UPDATE (AssistantPage.tsx)
========================================

Replace:

const reader = response.body.getReader();
const decoder = new TextDecoder();
let fullText = "";

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  const chunk = decoder.decode(value);
  fullText += chunk;
  setMessages(prev => updateLastAssistantMessage(fullText));
}
